## 4(a) Vanilla REINFORCE on CartPole-v1

### Network Architecture

An Actor-Critic network where the actor and value heads share the first layer:

| Layer | Details |
|-------|---------|
| Shared | `Linear(4 → 128)` + ReLU |
| Action head | `Linear(128 → 2)` + Softmax |
| Value head | `Linear(128 → 1)` |

All weights initialized with **Xavier uniform**. The network uses `float64` precision (`.double()`).

### Algorithm

Standard REINFORCE (Monte Carlo policy gradient):

1. Roll out a full episode (up to 9 999 steps).
2. Compute discounted rewards-to-go $G_t = \sum_{k=0}^\infty \gamma^k r_{t+k}$ backwards.
3. Normalize returns: $\hat G_t = (G_t - \mu_G) / (\sigma_G + \varepsilon)$.
4. Policy loss: $\mathcal L_\pi = -\sum_t \log\pi(a_t|s_t)\,\hat G_t$.
5. Value loss: $\mathcal L_V = \sum_t \mathrm{MSE}(V(s_t),\, G_t)$.
6. Total loss: $\mathcal L = \mathcal L_\pi + \mathcal L_V$. Back-propagate and update with Adam.

### Hyperparameters

| Parameter | Value |
|-----------|-------|
| Environment | `CartPole-v1` |
| Learning rate | 0.01 |
| Optimizer | Adam |
| $\gamma$ | 0.999 |
| EWMA $\alpha$ | 0.05 |
| Random seed | 10 |
| Max steps/episode | 9 999 |

### Results

Solved in **698 episodes** (EWMA reward 475.1, threshold 475.0).
Testing (10 episodes): **500 / 500** — perfect score on every episode.

## 4(b) REINFORCE with Baseline on LunarLander-v3

### Baseline Design

The **value function $V^{\pi_\theta}(s)$** (output of the shared value head) is used as the state-dependent baseline. The advantage estimate at each time step is:

$$A_t = G_t - V(s_t)$$

where $G_t$ is the discounted return computed from the trajectory.
The value is **detached** from the computation graph when computing the policy gradient (`.item()`), so gradients flow through the value network only via the separate MSE loss term.

**Why this baseline works:** Subtracting $V(s_t)$ leaves the expected gradient unchanged (standard baseline argument) while reducing variance because $V(s_t)$ is a good predictor of $G_t$, making the advantage $A_t$ much smaller in magnitude than the raw return.

### Network Architecture

Same shared architecture as (a), adapted for LunarLander-v3:

| Layer | Details |
|-------|---------|
| Shared | `Linear(8 → 128)` + ReLU |
| Action head | `Linear(128 → 4)` + Softmax |
| Value head | `Linear(128 → 1)` |

### Hyperparameters

|-----------|-------|
| Environment | `LunarLander-v3` |
| Learning rate | 0.002 |
| Optimizer | Adam |
| $\gamma$ | 0.99 |
| EWMA $\alpha$ | 0.05 |
| Random seed | 10 |

### Results

Solved in **10 411 episodes** (EWMA reward 202.6, threshold 200).
Testing (10 episodes): average reward **~258–311**.

## 4(c) REINFORCE with GAE on LunarLander-v3

### GAE Implementation

Generalized Advantage Estimation (Schulman et al., 2016) computes advantages via a **backwards recursive pass**:

$$\delta_t = r_t + \gamma \, V(s_{t+1})(1-\mathrm{done}_t) - V(s_t)$$

$$A_t^{\mathrm{GAE}} = \sum_{l=0}^{\infty} (\gamma\lambda)^l\, \delta_{t+l}
= \delta_t + \gamma\lambda(1-\mathrm{done}_t)\,A_{t+1}^{\mathrm{GAE}}$$

Key implementation details:
- **Bootstrap value:** if the episode ended due to **truncation** (timeout), $V(s_T)$ from the network is used as the bootstrap; if **terminated** (crash/landing), bootstrap = 0.
- The `values` list passed to `GAE.__call__` has length $T+1$ (including the bootstrap), so `values[t+1]` is always valid.
- `num_steps = None` → full-batch mode (entire episode used each update).
- Advantages are normalized before computing the policy loss.
- The **value network is supervised by discounted returns** $G_t$ (not by the GAE advantages) via MSE loss.

#### Bias–variance tradeoff

| $\lambda$ | Behavior |
|-----------|----------|
| $\lambda \to 0$ | High bias, low variance (approaches TD(0)) |
| $\lambda \to 1$ | Low bias, high variance (approaches Monte Carlo) |
| $\lambda = 0.95$ | Balanced; empirically strong default |

### Hyperparameters

| Parameter | Value |
|-----------|-------|
| Environment | `LunarLander-v3` |
| Learning rate | 0.005 |
| Optimizer | Adam |
| $\gamma$ | 0.99 |
| $\lambda$ | 0.90 / 0.95 / 0.99 |
| EWMA $\alpha$ | 0.05 |
| Random seed | 10 |

### Results

| $\lambda$ | Episodes to Solve | Final EWMA Reward |
|-----------|:-----------------:|:-----------------:|
| 0.90      | 14 928            | 203.0             |
| **0.95**  | **12 451**        | **202.2**         |
| 0.99      | 16 995            | 202.3             |

$\lambda = 0.95$ converges fastest. $\lambda = 0.90$ (more bias) and $\lambda = 0.99$ (more variance) both take substantially more episodes.
