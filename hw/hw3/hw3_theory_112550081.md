## Problem 1 (Soft Policy Improvement for Regularized MDPs) — 20 pts

We want to solve the constrained optimization

$$\max_{\pi(\cdot|s)} \sum_{a \in \mathcal{A}} \Big( \pi(a|s) Q_\Omega^{\pi_k}(s,a) - \pi(a|s) \log \pi(a|s) \Big), \quad \text{s.t.} \sum_{a \in \mathcal{A}} \pi(a|s) - 1 = 0.$$

Introduce a Lagrange multiplier $\mu \in \mathbb{R}$ for the simplex constraint. The Lagrangian is

$$L(\pi) = \sum_{a \in \mathcal{A}} \Big( \pi(a|s) Q_\Omega^{\pi_k}(s,a) - \pi(a|s) \log \pi(a|s) \Big) - \mu \Big( \sum_{a \in \mathcal{A}} \pi(a|s) - 1 \Big).$$

**Step 1 — Take the partial derivative w.r.t. each $\pi(a|s)$.** Using $\dfrac{d}{dx}(x\log x) = \log x + 1$:

$$\frac{\partial L(\pi)}{\partial \pi(a|s)} = Q_\Omega^{\pi_k}(s,a) - \big(\log \pi(a|s) + 1\big) - \mu.$$

**Step 2 — Set the derivative to zero.** The stationarity condition $\partial L / \partial \pi(a|s) = 0$ gives

$$\log \pi(a|s) = Q_\Omega^{\pi_k}(s,a) - 1 - \mu \;\Longrightarrow\; \pi(a|s) = \exp\!\big(Q_\Omega^{\pi_k}(s,a) - 1 - \mu\big).$$

**Step 3 — Enforce the simplex constraint.** Since $\sum_{a} \pi(a|s) = 1$,

$$\exp(-1-\mu) \sum_{a \in \mathcal{A}} \exp\!\big(Q_\Omega^{\pi_k}(s,a)\big) = 1 \;\Longrightarrow\; \exp(-1-\mu) = \frac{1}{\sum_{a' \in \mathcal{A}} \exp\!\big(Q_\Omega^{\pi_k}(s,a')\big)}.$$

**Step 4 — Substitute back.** We obtain the closed-form optimum (a *softmax* / Boltzmann policy):

$$\boxed{\;\pi_{k+1}(a|s) = \frac{\exp\!\big(Q_\Omega^{\pi_k}(s,a)\big)}{\sum_{a' \in \mathcal{A}} \exp\!\big(Q_\Omega^{\pi_k}(s,a')\big)}.\;}$$

The second-order test confirms a maximum: $\partial^2 L / \partial \pi(a|s)^2 = -1/\pi(a|s) < 0$, so $L$ is strictly concave and the stationary point is the unique global maximizer. $\blacksquare$

## Problem 2(a) — Actor Network of SAC (10 pts)

### (i) Why output the mean and the log-standard-deviation?

SAC parameterizes a **stochastic Gaussian policy** $\pi_\phi(a|s) = \mathcal{N}\!\big(\mu_\phi(s), \sigma_\phi(s)^2\big)$ for *continuous* actions, so the network must output both location ($\mu$) and scale ($\sigma$) parameters.

We predict $\log \sigma$ rather than $\sigma$ directly for two reasons:

1. **Positivity is automatic.** $\sigma = \exp(\log \sigma) > 0$ for any real-valued network output, so we do not need an extra activation (e.g. softplus or ReLU) that could cause vanishing gradients near zero.
2. **Numerical stability and wider dynamic range.** Standard deviations span many orders of magnitude (e.g. $10^{-9}$ to $10^{2}$). Operating in log-space yields better-conditioned gradients, and clamping $\log\sigma \in [-20, 2]$ (as in `sac_starter.py` lines 38–39, 63) prevents the distribution from collapsing to a delta function or exploding.

### (ii) How is the reparameterization trick implemented?

A naïve sampling $a \sim \mathcal{N}(\mu_\phi, \sigma_\phi^2)$ would be **non-differentiable** with respect to $\phi$, blocking gradients from the policy-loss back to the actor weights. The reparameterization trick rewrites the sample as a deterministic transformation of an independent noise variable:

$$a = \mu_\phi(s) + \sigma_\phi(s) \odot \epsilon, \qquad \epsilon \sim \mathcal{N}(0, I).$$

Now $a$ is a differentiable function of $\phi$, and the gradient $\nabla_\phi$ can be propagated through both $\mu_\phi$ and $\sigma_\phi$.

In `sac_starter.py` lines 67–68 this is realized by

```python
dist = Normal(mu, std)
z = dist.rsample()         # rsample() uses the reparam. trick
                           # equivalent to: z = mu + std * torch.randn_like(mu)
```

PyTorch's `rsample()` (as opposed to `sample()`) returns a *reparameterized* sample that preserves the computation graph, allowing gradient flow.

### (iii) Why is the log-probability adjusted (lines 72–74)?

The action that is actually applied to the environment is squashed to $(-1, 1)$ via the $\tanh$ transform:

$$a = \tanh(z), \qquad z \sim \mathcal{N}(\mu, \sigma^2).$$

Because $\tanh$ is an **invertible change of variables**, the density of the transformed variable must be corrected by the Jacobian (Appendix C of the original SAC paper):

$$\log \pi(a|s) = \log \mu(z|s) - \sum_{i=1}^{|\mathcal{A}|} \log\!\big(1 - \tanh(z_i)^2\big),$$

since $\dfrac{d a_i}{d z_i} = 1 - \tanh(z_i)^2$. The starter code

```python
action  = z.tanh()
log_prob = dist.log_prob(z) - torch.log(1 - action.pow(2) + 1e-7)
log_prob = log_prob.sum(-1, keepdim=True)
```

implements exactly this correction. Without it, the entropy term used in the SAC objective would be **biased** (computed in $z$-space instead of $a$-space) and the actor would learn a wrong policy. The `+ 1e-7` term prevents `log(0)` when actions saturate near $\pm 1$, and `.sum(-1)` aggregates over the action dimensions because each is conditionally independent.

## Problem 2(b) — Soft Q Network of SAC (10 pts)

### (i) Why two Q networks (the "twin Q" / clipped-double-Q trick)?

In standard $Q$-learning the target $y = r + \gamma \max_{a'} Q(s', a')$ contains a maximization over a noisy estimator, which produces a **positive (overestimation) bias** that compounds through bootstrapping and destabilizes training (Hasselt 2010; Fujimoto et al., *TD3*, 2018).

SAC adopts TD3's **Clipped Double-Q** remedy: maintain two independently-initialized critics $Q_{\theta_1}, Q_{\theta_2}$ and use their *minimum* whenever a Q-estimate is required as a bootstrapping target (e.g. in the V-loss or the actor loss):

$$\hat{Q}(s,a) = \min\big(Q_{\theta_1}(s,a), Q_{\theta_2}(s,a)\big).$$

Taking the minimum is a pessimistic correction that systematically counteracts the upward bias of the max-operator, yielding more accurate value estimates and substantially lowering the variance of the policy gradient.

### (ii) Loss function for the two soft Q networks

With the value-network target $V_{\bar{\psi}}$, the TD target is

$$y_i = r_i + \gamma\,(1 - d_i)\, V_{\bar{\psi}}(s'_i),$$

and each critic is trained with the mean-squared Bellman error:

$$\boxed{\;\mathcal{L}_{Q_j}(\theta_j) = \mathbb{E}_{(s_i,a_i,r_i,s'_i,d_i)\sim\mathcal{D}}\!\left[\frac{1}{2}\big( Q_{\theta_j}(s_i,a_i) - y_i \big)^2\right], \quad j = 1, 2.\;}$$

In mini-batch form (size $N$):

$$\mathcal{L}_{Q_j} = \frac{1}{N} \sum_{i=1}^{N} \Big( Q_{\theta_j}(s_i,a_i) - \big[r_i + \gamma\,(1 - d_i)\, V_{\bar{\psi}}(s'_i)\big] \Big)^2.$$

The target $y_i$ is *detached* from the computation graph (no gradient through $V_{\bar{\psi}}$).

## Problem 2(c) — Alpha Network of SAC (10 pts)

The enhanced SAC paper (Haarnoja et al., 2018; Sec. 5 of https://arxiv.org/abs/1812.05905) reformulates the maximum-entropy RL problem as a *constrained* optimization in which the average entropy of the policy must exceed a target $\mathcal{H}_0$ (chosen heuristically as $-\dim(\mathcal{A})$, see line 217 of the starter):

$$\max_{\pi_{0:T}} \;\mathbb{E}\!\left[\sum_{t} r_t\right] \quad\text{s.t.}\quad \mathbb{E}_{(s_t,a_t)\sim\rho_\pi}\!\big[-\log \pi(a_t|s_t)\big] \ge \mathcal{H}_0 \;\;\forall t.$$

Forming the Lagrangian with multiplier $\alpha \ge 0$ and applying dual descent, the *dual* problem (with the inner-policy optimum frozen) reduces, for each step, to

$$\min_{\alpha \ge 0}\; \mathbb{E}_{a_t \sim \pi_t^*}\!\big[-\alpha \log \pi_t^*(a_t|s_t) - \alpha\,\mathcal{H}_0\big] \;=\; \min_{\alpha \ge 0}\; \mathbb{E}_{a_t \sim \pi_t^*}\!\big[\,-\alpha\,\big(\log \pi_t^*(a_t|s_t) + \mathcal{H}_0\big)\big].$$

Optimizing in **log-space** ($\alpha = \exp(\log\alpha)$) automatically enforces positivity and improves numerical stability. With a minibatch estimate, the loss in line 286 of `sac_starter.py` is

$$\mathcal{L}(\alpha) = \frac{1}{N}\sum_{i}\Big[-\exp(\log\alpha) \cdot \big(\log \pi_\phi(a_i|s_i) + \mathcal{H}_0\big)\Big],$$

with $\log\pi_\phi(a_i|s_i)$ **detached** (the multiplier only adapts $\alpha$, not the policy).

**Intuition of the update.** Taking the gradient w.r.t. $\log\alpha$:

- If the current entropy $-\mathbb{E}[\log\pi] < \mathcal{H}_0$ (policy too deterministic), then $\log\pi + \mathcal{H}_0 > 0$, and the gradient *increases* $\alpha$ → stronger entropy bonus → more exploration.
- If the entropy is already above the target ($\log\pi + \mathcal{H}_0 < 0$), the gradient *decreases* $\alpha$ → less entropy weight → more greedy exploitation.

Thus $\alpha$ self-adjusts to keep the policy entropy near the target, removing the need to hand-tune one of SAC's most sensitive hyperparameters.
